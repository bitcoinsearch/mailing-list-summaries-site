Summary:

The bitcoin-dev mailing list is currently discussing various technical details of Bitcoin's sidechain technology and the proposed two-way pegged sidechain system, drivechains. The use of OP_BRIBE as a way to allow miners to be paid in exchange for their help in guaranteeing that a sidechain block is added to the largest chain is also discussed. The proposal for the process of transactions using OP_BRIBE_VERIFY involves sidechain miners broadcasting transactions with an OP_BRIBE output, which ends up in the memory pool of miners and is added to their next block. There is also discussion on the vulnerability of sidechains to theft and the need for restrictions on the amount of coin transferred from the side chain to the main chain.The Drivechain proposal, which enables a hybrid model where both a federation and miners provide acknowledgements of the sidechain state, is being supported by users, businesses, and developers. Another economic model being discussed is BMM (Bitcoin Meta Transactions), which is similar to Bitcoin's Proof-of-Work. In addition, there are conversations on the commitment structure for the coinbase transaction in drivechain implementation. In discussions about the security risks of implementing sidechains, concerns have been raised about the potential vulnerability of sidechains to theft, as a rich attacker could buy the entire reserve of a sidechain by paying the timeout multiplied by average transaction fees.
